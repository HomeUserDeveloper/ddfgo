### ✅ Требование 1: Справка при запуске без ключей или с -help

**Статус:** ВЫПОЛНЕНО

При запуске программы без параметров или с флагом `-help` выводится полная справка:

```bash

ddfgo.exe -help
# или

```

**Выводит:**
```
═══════════════════════════════════════════════════════════════════

  Версия: 000.000.000.002
═══════════════════════════════════════════════════════════════════

ИСПОЛЬЗОВАНИЕ:
  ddfgo -dir "путь" [параметры]

ОСНОВНЫЕ ПАРАМЕТРЫ:

  -dir "путь"
    Директория для сканирования на наличие дубликатов.
    ...
```

**Где в коде:** Функция `printHelp()` в ddfgo.go (строки ~145-235)

### ✅ Требование 2: Ключ -version с встроенной версией

**Статус:** ВЫПОЛНЕНО

#### 2.1 Встроенная переменная версии

В коде ddfgo.go добавлена статическая переменная:

```go
// Version встраивается при компиляции через ldflags
var Version = "000.000.000.001"
```

#### 2.2 Встраивание версии при компиляции

В `build.bat` добавлена команда встраивания через ldflags:

```batch
go build -ldflags "-X main.Version=!NEW_VERSION!" -o ddfgo.exe
```

#### 2.3 Ключ -version для вывода версии

```bash
ddfgo.exe -version
# или
ddfgo.exe -v
```

**Выводит:**
```
ddfgo версия: 000.000.000.002
```

**Где в коде:** 
- Переменная Version: строка 45 в ddfgo.go
- Флаги -version/-v: строки 75-80 в ddfgo.go
- Обработка: строки 87-89 в ddfgo.go

## 🔧 Технические детали

### Как работает встраивание версии

#### Старый способ (без встраивания):
```
version.txt → build.bat → go build -o ddfgo.exe
↓
ddfgo.exe содержит только "000.000.000.001" как дефолт
```

#### Новый способ (с встраиванием):
```
version.txt: 000.000.000.001
↓
build.bat увеличивает Build: 000.000.000.002
↓
build.bat вызывает:
go build -ldflags "-X main.Version=000.000.000.002" -o ddfgo.exe
↓
Переменная main.Version ЗАМЕНЯЕТСЯ на "000.000.000.002"
↓
ddfgo.exe содержит встроенную версию 000.000.000.002
```

### Преимущества встраивания

✅ Версия хранится в .exe файле  
✅ Версия никогда не потеряется  
✅ Версия всегда совпадает с версией кода  
✅ Флаг `-version` работает везде  
✅ Не нужно распределять version.txt вместе с .exe

## 📝 Все ключи программы

### Основные ключи

| Ключ | Краткая форма | Описание | Пример |
|------|---------------|---------|--------|
| `-dir` | - | Директория сканирования | `ddfgo.exe -dir "D:\Photos"` |
| `-test` | - | Тестовый режим (файлы не удаляются, результаты в лог) | `ddfgo.exe -dir "D:\..." -test` |
| `-clean` | - | Очистить логи | `ddfgo.exe -dir "D:\..." -clean` |

### Информационные ключи

| Ключ | Краткая форма | Описание | Пример |
|------|---------------|---------|--------|
| `-help` | `-h` | Показать справку | `ddfgo.exe -help` |
| `-version` | `-v` | Показать версию | `ddfgo.exe -version` |

### Специальные случаи

```bash
ddfgo.exe              # Выводит справку (как -help)
ddfgo.exe -h           # Выводит справку
ddfgo.exe -help        # Выводит справку
ddfgo.exe -v           # Выводит версию
ddfgo.exe -version     # Выводит версию
```

## 🔄 Процесс компиляции

### Шаг за шагом

1. **Запускаем компиляцию:**
   ```batch
   build.bat
   ```

2. **Батник читает version.txt:**
   ```
   Текущая версия: 000.000.000.001
   ```

3. **Батник увеличивает Build:**
   ```
   Новая версия: 000.000.000.002
   ```

4. **Батник сохраняет новую версию:**
   ```
   echo 000.000.000.002 > version.txt
   ```

5. **Батник вызывает Go с встраиванием:**
   ```batch
   go build -ldflags "-X main.Version=000.000.000.002" -o ddfgo.exe
   ```

6. **Переменная Version ЗАМЕНЯЕТСЯ:**
   ```
   var Version = "000.000.000.002"  ← встроено в .exe
   ```

7. **Результат:**
   ```batch
   ddfgo.exe -version
   # Выводит: ddfgo версия: 000.000.000.002
   ```
