КРАТКОЕ ОПИСАНИЕ
================================================================================

ddfgo - высокопроизводительная утилита поиска и удаления дубликатов файлов
на Windows, оптимизированная для работы с большими наборами данных (100K+ файлов).

**Текущая версия включает:**
- Priority 1: Адаптивное хеширование и распределение воркеров (+23.5% производительность)
- Priority 2: Оптимизация памяти, потоковая SQL-фильтрация дубликатов
- **Конечный результат: +23.5% улучшение производительности на реальных данных**

ИСПОЛЬЗОВАНИЕ
================================================================================

ОСНОВНОЙ СИНТАКСИС:
  ddfgo.exe -dir "путь_к_директории" [опции]

ОСНОВНЫЕ ОПЦИИ:
  -dir "путь"         Директория для сканирования (обязательно)
  -test               Тестовый режим: файлы не удаляются, результаты записываются в лог
  -clean              Очистить лог перед запуском
  -all                Удалять все дубликаты (включая < 10KB)
  -dir0               Удалить пустые каталоги после завершения
  -ext-all            Обрабатывать файлы со всеми расширениями
  -help, -h           Показать справку
  -version, -v        Показать номер версии

ПРИМЕРЫ:
  # Сканирование на наличие дубликатов
  ddfgo.exe -dir "D:\Downloads" -test -clean
  
  # Удаление дубликатов всех размеров
  ddfgo.exe -dir "D:\Photos" -all
  
  # С очисткой пустых каталогов
  ddfgo.exe -dir "D:\Archive" -dir0

================================================================================
ТЕХНИЧЕСКИЕ ДЕТАЛИ
================================================================================

ОПТИМИЗАЦИЯ БЫСТРОГО ХЕША (calculateQuickHashBlockSize):
  • Маленькие файлы (< 100 KB): весь файл
  • Маленькие (100 KB - 1 MB): 10%
  • Средние (1 MB - 100 MB): 5%
  • Большие (100 MB - 1 GB): 2%
  • Очень большие (1+ GB): 1%, max 10 MB

ОПТИМИЗАЦИЯ ВОРКЕРОВ (calculateOptimalWorkerCount):
  • < 100 файлов: 1 воркер
  • < 1,000: numCPU
  • < 100,000: numCPU * 2
  • < 1,000,000: numCPU * 4
  • ≥ 1,000,000: min(numCPU * 8, 256)

ПОТОКОВАЯ SQL-ФИЛЬТРАЦИЯ:
  • Вместо загрузки всех дубликатов в памяти G используется SQL streaming
  • Создается временная таблица для фильтрации
  • Fast-path при отсутствии дубликатов (одна DELETE операция)
