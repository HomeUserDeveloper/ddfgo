# Флаг -keep-old и поведение по умолчанию в ddfgo

## Описание

Начиная с текущей версии, ddfgo по умолчанию сохраняет самый новый файл в группе дубликатов.
Для обратного поведения используется флаг `-keep-old`.

### Поведение по умолчанию (без флага)
- **Сохраняет:** самый НОВЫЙ файл (последнее обновление)
- **Удаляет:** старые версии (дубликаты)
- **Сортировка:** `ORDER BY fmod DESC` (время модификации в порядке убывания)

### С флагом -keep-old
- **Сохраняет:** самый СТАРЫЙ файл (первый созданный)
- **Удаляет:** более новые версии (дубликаты)
- **Сортировка:** `ORDER BY fmod ASC` (время модификации в порядке возрастания)

## Примеры использования

### Пример 1: Поведение по умолчанию (оставить последнюю версию)
```bash
ddfgo -dir "D:\Projects" -test -clean
```
Сохранит самый новый (последний обновленный) файл, удалит все старые версии.

### Пример 2: Очистка полной папки версий
```bash
ddfgo -dir ".\testdir" -all -ext-all -dir0
```
- Сохранит только самые новые версии файлов
- Удалит все старые дубликаты
- Удалит пустые каталоги (флаг `-dir0`)

### Пример 3: Обратный режим (оставить самый старый)
```bash
ddfgo -dir "E:\Backups" -keep-old -test
```
Запустит обратный режим: сохранит самый старый файл и удалит новые копии.

## Сценарии использования

### Сценарий 1: Очистка архивов версий
Если у вас есть копии файлов с разными временами создания, а вы хотите оставить только самую новую:

```
folder/
├── v1.0/file.txt (2026-01-01 10:00)
├── v1.1/file.txt (2026-02-01 10:00)
└── v1.2/file.txt (2026-04-01 10:00)  ← Будет сохранен по умолчанию
```

### Сценарий 2: Удаление резервных копий
Когда есть несколько резервных копий данных, оставить только самую свежую:

```bash
ddfgo -dir "D:\Backups" -all -dir0
```

## Технические детали

Режимы работают путем изменения порядка сортировки в SQL запросе:

**Без флага (ORDER BY fmod DESC, fname DESC):**
- ROW_NUMBER() = 1 → самый новый файл (сохраняется)
- ROW_NUMBER() > 1 → старые версии (удаляются)

**С флагом `-keep-old` (ORDER BY fmod ASC, fname ASC):**
- ROW_NUMBER() = 1 → самый старый файл (сохраняется)
- ROW_NUMBER() > 1 → новые версии (удаляются)

## Предупреждения

- Выбор файла для сохранения работает на основе времени модификации (`mtime`)
- Если все файлы имеют одинаковое время, они сортируются по имени пути (лексикографически)
- В режиме `-test` можно проверить какие файлы будут удалены перед реальным удалением
