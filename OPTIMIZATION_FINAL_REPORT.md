   - Логика: адаптирует размер блока в зависимости от размера файла
   - Стратегия:

     * Очень большие (> 1 GB): 1%
   - Преимущества:
     ✓ Лучшая точность на больших файлах
     ✓ Быстрее на маленьких файлах
     ✓ Баланс точность/скорость
   
   ✅ ОПТИМИЗАЦИЯ 2: Адаптивное количество воркеров
   ────────────────────────────────────────────────────
   - Функция добавлена: calculateOptimalWorkerCount()
   - Параметры расчета: количество файлов для обработки
   - Стратегия:
     * < 100 файлов: 1 воркер
     * < 1,000 файлов: numCPU
     * < 100,000 файлов: numCPU * 2
     * < 1,000,000 файлов: numCPU * 4
     * >= 1,000,000 файлов: numCPU * 8 (max 256)
   - Преимущества:
     ✓ Оптимально использует I/O для больших наборов
     ✓ Не тратит память на мало файлов
