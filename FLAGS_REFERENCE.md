## Основные ключи

### `-dir "путь"` - Директория для сканирования
Обязательный параметр (если не используется переменная окружения DDF_DIR)

```batch
ddfgo.exe -dir "D:\Downloads"
ddfgo.exe -dir "E:\Photos\MyPhotos"
```

**Поведение:**
- Если не указан, ищет переменную окружения `DDF_DIR`
- Если переменная не установлена, используется `E:\SortDir`

### `-test` - Тестовый режим (без удаления файлов)
Включает логирование результатов БЕЗ удаления файлов

```batch
ddfgo.exe -dir "D:\Downloads" -test
```

**Результат:**
- Создает файл `ddfgo.log` с результатами
- Файлы НЕ удаляются
- Можно проверить результаты перед реальным запуском

### `-clean` - Очистить старые логи
Удаляет старые лог файлы перед запуском

```batch
ddfgo.exe -dir "D:\Downloads" -test -clean
```

**Результат:**
- Удаляет `ddfgo.log` и `ddfgoerr.log` (если были)
- Создает новые чистые логи
- Используется с флагом `-test`

### `-cpuprofile "файл"` - Сохранить CPU профиль
Включает встроенное CPU-профилирование выполнения приложения

```batch
ddfgo.exe -dir "D:\Downloads" -test -cpuprofile cpu.prof
```

**Результат:**
- Создается файл профиля CPU
- Профиль можно открыть через `go tool pprof`

### `-memprofile "файл"` - Сохранить профиль памяти
Сохраняет heap-профиль по завершении запуска

```batch
ddfgo.exe -dir "D:\Downloads" -test -memprofile mem.prof
```

**Результат:**
- Создается файл профиля памяти
- Профиль можно открыть через `go tool pprof`

### `-help` или `-h` - Показать справку
Выводит полную справку по использованию

```batch
ddfgo.exe -help
ddfgo.exe -h
```

**Результат:**
- Выводит справку в консоль
- Показывает все доступные параметры
- Показывает примеры использования

### `-version` или `-v` - Показать версию
Выводит номер версии программы

```batch
ddfgo.exe -version
ddfgo.exe -v
```

**Результат:**
```
ddfgo версия: 000.000.000.002
```

### Запуск БЕЗ параметров
При запуске без ключей выводится справка

```batch
ddfgo.exe
```

**Результат:** То же как и `-help`

## Примеры использования

### Пример 1: Показать справку
```batch
REM Все эти команды показывают справку:
ddfgo.exe
ddfgo.exe -help
ddfgo.exe -h
```

### Пример 2: Показать версию
```batch
REM Все эти команды показывают версию:
ddfgo.exe -version
ddfgo.exe -v
```

**Вывод:**
```
ddfgo версия: 000.000.000.002
```

### Пример 3: Простое сканирование
```batch
ddfgo.exe -dir "D:\Downloads"
```
**Действие:** Найдет и удалит дубликаты в папке Downloads

### Пример 4: Безопасное сканирование (с проверкой)
```batch
REM Сначала тестируем
ddfgo.exe -dir "D:\Photos" -test -clean

REM Проверяем результаты
type ddfgo.log

REM Если результаты корректны, запускаем реально
ddfgo.exe -dir "D:\Photos"
```

### Пример 5: С переменной окружения
```batch
REM Установим переменную окружения
set DDF_DIR=D:\MyFiles

REM Теперь можно запустить просто:
ddfgo.exe

REM Или с параметрами:
ddfgo.exe -test -clean
```

### Пример 6: Комбинированный запуск
```batch
ddfgo.exe -dir "E:\Videos" -test -clean
```
**Действие:**
1. Удаляет старые логи
2. Сканирует папку E:\Videos
3. Логирует результаты в ddfgo.log
4. НЕ удаляет файлы

### Пример 7: Запуск с профилированием
```batch
ddfgo.exe -dir "D:\Data" -test -clean -cpuprofile cpu.prof -memprofile mem.prof

REM Анализ профилей:
go tool pprof ddfgo.exe cpu.prof
go tool pprof ddfgo.exe mem.prof
```

## Встроенная версия

### Как версия встраивается в код

При компиляции через `build.bat` версия встраивается в исполняемый файл:

```batch
REM build.bat выполняет:
go build -ldflags "-X main.Version=000.000.000.002" -o ddfgo.exe
```

**Преимущества:**
- Версия хранится в исполняемом файле
- Не нужно искать файл `version.txt` при распределении бинарника
- Версия всегда совпадает с версией кода (build.bat синхронизирует)
- Флаг `-version` показывает встроенную версию

### Как это работает в коде

В файле `ddfgo.go`:

```go
// Version встраивается при компиляции через ldflags
var Version = "000.000.000.001"
```

При компиляции переменная заменяется на текущую версию.

### Проверка встроенной версии

```batch
REM Посмотри встроенную версию:
ddfgo.exe -version

REM Вывод:
REM ddfgo версия: 000.000.000.002
```

## 🔄 Процесс компиляции и версионирования

### Раньше (старая система)
```
version.txt: 000.000.000.001
build.bat → go build
↓ Версия НЕ встроена в исполняемый файл
ddfgo.exe (версия только в version.txt)
```

### Теперь (новая система)
```
version.txt: 000.000.000.001
↓
build.bat читает версию и увеличивает Build
↓
version.txt: 000.000.000.002
↓
build.bat компилирует с -ldflags "-X main.Version=000.000.000.002"
↓
ddfgo.exe (версия встроена в исполняемый файл!)
↓
ddfgo.exe -version → "ddfgo версия: 000.000.000.002"
```

**Преимущества:**
- ✅ Версия встроена в исполняемый файл
- ✅ Версия всегда совпадает
- ✅ Не нужна версия при распределении
- ✅ Флаг `-version` работает везде

## 📝 Таблица всех флагов

| Флаг | Краткая форма | Назначение | Пример |
|------|--------------|-----------|--------|

## ⚠️ Важные замечания

### Запуск БЕЗ параметров
```batch
ddfgo.exe
```
Выводит справку (как `ddfgo.exe -help`)

### Запуск с неправильными параметрами
```batch
ddfgo.exe -unknown-flag
```
Go флаг парсер покажет ошибку и справку

### Версия всегда встроена
```batch
# Даже если удалите version.txt, -version все равно работает:
ddfgo.exe -version
# Выведет встроенную версию
```
